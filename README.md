## Vehicle Health Checker — Интеллектуальная система контроля технического состояния автомобиля
Система автоматизирует учет износа автомобильных деталей, рассчитывая остаточный ресурс расходников на основе реального пробега транспортного средства и индивидуальных регламентов заводов-изготовителей.

## Проблема
Основная проблема автовладельцев — отсутствие системного контроля технического состояния автомобиля. Расходные материалы (моторное масло, тормозные колодки, свечи зажигания) изнашиваются незаметно для водителя. Существующие методы контроля неэффективны: бумажные сервисные книжки часто теряются, а стандартные напоминания в календарях учитывают только прошедшее время, игнорируя реальный пробег и специфику эксплуатации конкретного типа автомобиля (например, разницу в износе деталей у внедорожников и малолитражных машин). В результате автовладельцы пропускают сроки обслуживания, что приводит к критическим поломкам, дорогостоящему ремонту двигателя или ходовой части, а также снижает безопасность дорожного движения.

## Решение
Vehicle Health Checker представляет собой цифровой сервис, который полностью решает проблему своевременного обслуживания. Пользователь вносит данные о текущем пробеге автомобиля, после чего система автоматически сопоставляет эти сведения с историей предыдущих замен и техническим регламентом для данного класса авто. Алгоритм выявляет критические отклонения от нормы, прогнозирует износ и выдает пользователю структурированные рекомендации о необходимости проведения технического обслуживания.

## Ключевые возможности
Автоматизированный расчет износа: Система вычисляет разницу между текущим пробегом и пробегом на момент последней замены детали, присваивая текущему состоянию статус (OK, Warning, Critical).

Индекс технического состояния: Математическая оценка общей исправности автомобиля по 100-балльной шкале, которая снижается пропорционально количеству просроченных регламентных работ.

Аргументированные рекомендации: Система предоставляет пользователю точные данные о том, какие компоненты требуют замены и на сколько километров превышен безопасный лимит эксплуатации.

Каскадное управление данными: Поддержка ведения нескольких автомобилей в одном профиле. При удалении транспортного средства из системы автоматически и безвозвратно очищается вся связанная с ним история ремонтов на уровне базы данных.

## Технический стек
Бэкенд: Python (FastAPI — асинхронная обработка запросов).

Слой данных (ORM): SQLAlchemy ORM (трансляция объектной логики Python в реляционную структуру базы данных).

База данных: MySQL Server 8.0 (реляционная СУБД со строгой типизацией данных, индексацией полей и контролем целостности через внешние ключи).

Архитектурный подход: Объектно-Ориентированное Программирование (ООП).

## Бизнес-логика и методы классов (Бэкенд)
Логика работы приложения инкапсулирована в методах соответствующих программных классов:

1. Управление аккаунтами (Класс User)
register_user() — Создание профиля нового пользователя, хэширование пароля и сохранение учетных данных в базу.

login_user() — Проверка подлинности учетных данных при входе на сайт и инициализация сессии.

logout_user() — Завершение текущей сессии пользователя.

2. Управление транспортными средствами (Класс Car)
add_car() — Регистрация автомобиля в профиле пользователя с указанием марки, модели, типа ТС и начального пробега.

update_mileage() — Обновление текущего пробега автомобиля. Данный метод является триггером для запуска диагностического алгоритма.

remove_car() — Удаление автомобиля из профиля пользователя с активацией каскадного удаления связанных данных в СУБД.

3. Учет сервисного обслуживания (Класс ServiceHistory)
add_service_record() — Фиксация факта замены конкретной детали с указанием даты и пробега на момент ремонта. Метод обнуляет счетчик износа для обслуженного компонента.

get_car_history() — Формирование и вывод хронологического отчета обо всех выполненных ремонтных работах для выбранного автомобиля.

4. Диагностический модуль (Класс DiagnosticEngine)
run_diagnostic() — Вычислительный алгоритм, сопоставляющий текущую разницу пробега с установленными лимитами из справочника регламентов.

generate_recommendations() — Формирование текстовых предупреждений и инструкций для автовладельца на основе полученного статуса износа.

save_log() — Запись результатов проведения проверки в архивную таблицу логов для последующего анализа.
